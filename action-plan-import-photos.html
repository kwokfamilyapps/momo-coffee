3. 💡 建議方案：Folder + 時間每 3 張一組

規則：

  1. 先按來源 Folder 分開（郊遊徑 / 清理水道 / 山徑巡查）
  2. 每個 Folder 內，相同 GPS CheckPoint 嘅相按時間排序
  3. 每 3 張斬一組，獨立成一條 record
  4. 每條 record 用該組第一張相嘅 GPS 坐標
  5. 每條 record 帶 3 張 resize 後嘅 base64 相片

✅ 預計結果：~44 條 records
郊遊徑水道 18張 → ~10 records
清理水道 24張 → ~16 records
山徑巡查 30張 → ~18 records

4. 🔧 執行步驟

  1. 下載 72 張 HEIC — 經 Google Drive API download
  2. 讀取 EXIF — 抽出 DateTime + GPS + 海拔
  3. Resize 相片 — HEIC → JPEG 1024px 長邊，quality 80%，~100-200KB each
  4. 分組 — 按 Folder → 按 CheckPoint → 按時間每 3 張一組
  5. 建立 Firestore records — 用 db.collection('records').add()
  6. 插入相片 — 用 db.collection('records').doc(id).update({ photosData })
  7. 匯出 log — 列出每條 record 嘅 ID、坐標、相片數

4.1 Technical Details

項目做法
HEIC 處理pillow-heif library → PIL Image
Resizemax 1024px（長邊）, JPEG quality 80
Firestore Auth用 API key（同 app_firebase.html 一樣嘅 config）
Upload 方法逐條 record add() → 再 update photosData
GPS 儲存{ lat: 22.397, lng: 114.249 }
時間格式Firestore Timestamp（由 EXIF datetime 轉）
