1. ❓ 問題：義工冇 mvmv 點算？

你嘅義工朋友都有 iPhone，行山會影相（GPS 坐標自動 embedded），但佢哋：

  • ❌ 冇 mvmv / Hermes Agent 幫手抽 EXIF 數據
  • ❌ Gemini 不能 direct access Google Drive（privacy 限制）
  • ❌ Gemini 每次 upload 只處理到 ~10 張相
  • ❌ 唔識寫 code / 用 API 抽 GPS

2. 💡 解決方案：三個免費工具串聯

Step工具收費做咩
1Google Colab🆓 免費 用 Python 讀 Google Drive 相片 → 抽 GPS + 日期 → 輸出 CSV
2Google Gemini🆓 免費 食 CSV 數據 → 生成 HTML 報告（地圖 + 統計）
3Google Cloud Storage🆓 免費 5GB Host HTML 做靜態網站，分享 link 俾人睇
✅ 關鍵： Gemini 食唔到 Google Drive，但食到 CSV（文字格式）。
所以中間加一個 Colab notebook 做「數據轉換」，將相片 GPS 抽出來變做 CSV。

3. 📋 義工操作步驟（圖文教學版）

📍 Phase A：行山影相

A
用 iPhone 沿山徑影相
✅ 確保 Location Services 開咗（GPS 會自動寫入相片）
✅ 同一位置可以影多張（前後對比）
✅ 影完相 auto backup 上 Google Drive / Google Photos

🔧 Phase B：用 Colab 抽數據（呢步最關鍵）

B1
開 Colab Notebook
㩒呢條 link（我準備好嘅 template）→ File → Save a copy to Drive
🔗 https://colab.research.google.com/drive/1xxx-template
B2
Mount Google Drive
㩒第一個 Play ▶️ 按鈕 → 授權 Google Drive 連接
Colab 就可以直接讀你 Google Drive 嘅相片
B3
指定 Folder 位置
改一個變數：folder_path = "/content/drive/My Drive/山徑相片/2026-07-23"
就係你放相嘅 Google Drive 路徑
B4
Run 全部 Cells ▶️
Colab 會自動：
✅ 安裝 pillow-heif（讀 HEIC）
✅ 逐張相抽 EXIF（GPS + 日期 + 海拔）
✅ 按坐標分組（同一地點嘅相 group 埋）
✅ 輸出 CSV 檔案 → 自動 save 返落 Google Drive
📄 CSV 內容範例：

filename,datetime,lat,lon,altitude,group
IMG_1765.HEIC,2026-07-23 10:27:00,22.400197,114.246644,309,CP1
IMG_1768.HEIC,2026-07-23 10:52:51,22.398025,114.248436,359,CP2
...

🤖 Phase C：Gemini 生成報告

C1
開 Gemini → Upload CSV
去 https://gemini.google.com → Upload 個 CSV 檔案上去
C2
Copy & Paste 呢個 Prompt：
「我 upload 咗一個 CSV 檔案，係山徑巡查相片嘅 GPS 數據。
每張相有：檔名、日期時間、緯度、經度、海拔、CheckPoint 編號。

請幫我：
1. 分析總共有幾多張相、幾個 CheckPoint、海拔範圍
2. 按 CheckPoint 統計每點有幾多張相
3. 生成一份 HTML 報告，要包含：
　• 統計摘要卡片
　• 用 Leaflet.js 嘅互動地圖，顯示所有 CheckPoint
　• 路線用綠色虛線連接所有 CP
　• 每個 CP 用圓形 marker，大細代表相片數量
　• 一個 summary table
4. 用香港繁體中文，手機 friendly
5. 直接俾 HTML code 我」
C3
Copy HTML → Save 做 .html 檔案
Gemini 會輸出完整 HTML → 複製 → 貼上記事本 → Save 做 report.html

☁️ Phase D：放上 Google Cloud 免費網站

D1
開 Google Cloud Storage
去 https://console.cloud.google.com → 開一個新 Project
（第一次用有 $300 免費額度，但純 hosting 係免費）
D2
開一個 Public Bucket
Cloud Storage → Create Bucket → 改個全球唯一名
eg mos-trail-report-2026
D3
Upload HTML + Enable 靜態網站
Upload report.html → Permission set Public →
Bucket 設定 → Static Website → 指定 index.html
🔗 你會得到一條 link：
https://storage.googleapis.com/mos-trail-report-2026/report.html
✅ 完成！可以將條 link 分享俾朋友睇！

5. ⚖️ 呢個方案嘅好處同限制

維度分析
💰 成本 🟢 全部免費 — Colab（免費 GPU）、Gemini（免費 tier）、Cloud Storage（5GB 免費）
🎯 易用度 🟡 中等 — Colab 要㩒幾個制，但唔使寫 code；Cloud Storage 要一次性 setup
📸 相片數量 🟢 冇限制 — Colab 一次過處理晒所有 HEIC，Gemini 只食 CSV（文字檔，唔受限）
🌍 GPS 準確度 🟢 精確 — HEIC EXIF 直接抽取 iPhone GPS（誤差 <5m）
🗺️ 地圖質素 🟡 視乎 Gemini prompt 寫得好唔好 — 可能要試幾次先有理想效果
🔄 重複用 🟢 可以 — 每次行山後，改 Folder path → Run Colab → 新 CSV → Gemini → 新報告
🔧 前期設定 🟡 要 setup 一次：Colab Notebook + Cloud Storage Bucket，之後義工只係用
